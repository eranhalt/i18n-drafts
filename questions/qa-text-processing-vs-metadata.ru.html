Типы объявления языка

В вебе всегда важно связать веб-контент с информацией о языке. С одной стороны, это важно для того чтобы веб-контент был правильно обработан или представлен читателю, с другой - также важно знать язык(и) целевой аудитории для которой предназначен веб-ресурс как целое. Это две разные вещи: технология должна предоставить отдельные способы для выражения каждой из них, а также создатели веб-контента должны использовать эти способы по назначению.

Эта статья описывает различие между следующими типами языковой информации: метаданные и обработка текста.

Язык обработки текста

Браузер или приложение обычно обязаны принять язык во внимание при отображении или управлении веб-контентом. Это включает в себя случаи как воспроизведения текста в браузере, запуск проверки орфографии, стилизирование переносов линии, использование дефисов, выбор шрифтов по умолчанию и многое другое требующие соответствующего использования языка. По этой причине, браузер должен знать конкретно с каким языком он работает для определенной части текста.

Итак, мы по необходимости говорим о связывании конкретного языка с текстом или некоторой частью текста в границах веб-ресурса. Несмотря на то что целевая аудитория может владеть более чем одним языком, определенная часть текста может иметь только один язык.

В HTML файле lang аттрибут использован для определения языка обработки текста. Он может быть использован для всей страницы а также для фрагментов страницы где язык меняется.

Необходимость конкретности влияет на то как язык должен быть объявлен для обработки текста. Поэтому lang аттрибут позволяет использоание только одного языкавого тэга.

Метаданные: язык целевой аудитории

Метаданные описывающие язык или языки целевой аудитории относятся к целому документу/веб-ресурсу. Такие метаданные могут быть использованы для поиска, подачи правильной языковой версии, управления процессом, классификации и т.д.

Язык целевой аудитории не требует упоминания каждого языка использованного в документе. Многие документы в вебе содержат в себе встроенные фрагменты веб-контента на разных языках, где страница сама по себе явно нацелена на носителей одного конкретного языка. Например, немецкий путеводитель по Пекину может содержать полезные фразы на китайском, но он ориентирован на немецкоязычную, а не на китайскую аудиторию.

С другой стороны, веб-страница может содержать в себе один и тот же либо параллельный контент на нескольких языках. Например, канадская веб-страница может приветствовать читателей контентом на французском языке в левой колонке и тем же самым контентом на английском в правой колонке. Таким образом документ равнозначно направлен на носттелей обеих языков, следовательно двух языковых целевых аудиторий. Эта ситуация не так распространена в Интернете, как в печатных материалах, поскольку легко сделать ссылку на отдельные страницы в Интернете для разных аудиторий, но это действительно происходит там, где есть многоязычные сообщества. Другой вариант использования - это блог или новостная страница, ориентированная на многоязычное сообщество, где некоторые статьи на странице написаны на одном языке, а некоторые - на другом. Например, форум, используемый сообществом пенджабцев, может содержать сообщения на английском, хинди и панджаби в одной ветке.

Есть также страницы, на которых навигационная информация, включая заголовок, написана на одном языке, а реальное содержание страницы - на другом. Хотя это не обязательно является хорошей практикой, это не меняет того факта, что язык целевой аудитории обычно является языком контента, независимо от языка в верхней части источника документа.

Для HTML-страницы метаданные об аудитории могут быть выражены в формате HTTP Content-Language заголовок. Этот заголовок содержимого может принимать несколько значений.

HTML страницы иногда включают meta element that can declare language in a similar way, eg. <meta http-equiv="content-language" content="en, fr"/>, но эта конструкция устарела и не должна использоваться. (Подробнее см. HTTP headers, meta elements and language information.)

Вывод языка обработки текста из метаданных

В некоторых случаях можно определить язык обработки текста по метаданным ресурса, но не всегда.

Если значение метаданных представляет собой список из более чем одного языка, должен быть способ определения, какой язык использовать при обработке контента.

Кроме того, при изменении языка внутри документа информация о языке целевой аудитории не может быть связана с соответствующей частью страницы или документа, как это было бы необходимо для обработки текста (т. Е. Таким образом, чтобы необходим для правильного применения языкового преобразования текста в речь, стиля, автоматического назначения шрифта и т. д. для различных частей документа.)

Поэтому при разработке новой технологии или формата данных разработчики должны предоставлять отдельные методы для выражения языка целевой аудитории и языка обработки текста.

Разработчики контента должны правильно использовать доступные конструкции.
